# v1.1 多端支持设计文档

日期:2026-07-16
状态:已评审(修订:微信小游戏 3D 升级为正式交付物)
范围:Web / Telegram 小程序 / macOS / Windows / iOS / Android / 微信小游戏(完整 3D)/ 微信小程序(无头层)

## 1. 目标与非目标

**目标**:让基于 `@overworld-engine/*` 的游戏用同一套代码覆盖上述所有端;
框架层提供平台检测、能力桥接与微信适配器;仓库提供四个可直接复制的端模板。

**非目标**:
- 不做 React Native 渲染路线(R3F native + DOM HUD 重写成本过高,收益低于 WebView 方案)。
- 微信**小程序**(非小游戏)不跑 3D(无 DOM,WXML 混排限制多),定位为
  "无头引擎 + 原生 WXML UI"(guides 提供接法);**完整 3D 由微信小游戏正式承诺**(见 §4.1)。
- 不做各应用商店的上架自动化(签名/公证/审核材料以文档形式给出)。

## 2. 平台矩阵与总体策略

一套 Web 代码,三类交付:

| 类别 | 端 | 交付方式 | 框架改动 |
|---|---|---|---|
| A:浏览器直达 | Web、**Telegram 小程序** | 就是网页;TG 只差初始化/视口/返回键接线 | 仅 platform 桥 |
| B:WebView 壳 | **macOS/Windows**(Tauri 2)、**iOS/Android**(Capacitor 7) | 壳内跑同一 web 构建 | platform 桥 + 壳模板 |
| C:适配层 | **微信小游戏**(完整 3D,正式支持)、**微信小程序**(无头层) | weapp-adapter + 注入式适配器 + R3F createRoot | adapters-weapp 包 + scene 跨端标签 |

策略核心:**框架不为任何端引入硬依赖**。所有端差异收敛到两个新包
(`platform` 桥接口 + `adapters-weapp`),壳 SDK(@tauri-apps/api、@capacitor/*、
Telegram WebApp)一律由**壳模板**持有,框架包通过运行时检测 + 动态探测消费,
保持 18 个既有包的依赖面不变。

## 3. 新包 ①:@overworld-engine/platform

运行时平台检测与能力桥。依赖:仅 `@overworld-engine/core`(peer:react 可选)。

### 3.1 检测与能力

```ts
type PlatformKind = 'web' | 'telegram' | 'tauri' | 'capacitor' | 'weapp' | 'node'
detectPlatform(): PlatformKind
// 检测顺序(特异性降序):wx(weapp)→ window.__TAURI_INTERNALS__(tauri)
// → window.Capacitor(capacitor)→ window.Telegram?.WebApp?.initData(telegram)
// → window(web)→ node。可被 configurePlatform({ force }) 覆盖(测试/调试)。

interface PlatformCapabilities {
  kind: PlatformKind
  hasDOM: boolean; hasWebGL: boolean
  hasTouch: boolean; hasKeyboard: boolean
  persistentStorage: 'localStorage' | 'file' | 'wx' | 'memory'
}
getCapabilities(): PlatformCapabilities
shouldShowTouchControls(): boolean   // 给 VirtualJoystick 的默认开关
recommendedQualityPreset(): 'high' | 'medium' | 'low'
// 平台修正版画质推荐:委托 scene 的启发式思想但不依赖 scene 包——
// 自带同款启发式 + 平台修正(telegram/capacitor 至多 medium)。
// 游戏侧:useQualityStore.setPreset(recommendedQualityPreset())
```

### 3.2 PlatformBridge 接口与内置桥

```ts
interface PlatformBridge {
  kind: PlatformKind
  /** 存档存储(满足 core 的 EnumerableStorage,直接喂 persistOptions/createSaveSlots) */
  storage(): EnumerableStorage
  openExternal(url: string): void          // 外链:TG 用 openLink,壳用系统浏览器
  safeAreaInsets(): { top: number; right: number; bottom: number; left: number }
  vibrate?(pattern: 'light' | 'medium' | 'heavy'): void
  /** 生命周期接线:实现负责监听端事件并在总线上发 app:* 事件,返回解绑 */
  bindLifecycle(bus: EventBus<OverworldEventMap>): () => void
  quit?(): void                            // 桌面壳可用
}
createBridge(kind?): PlatformBridge        // 缺省按 detectPlatform()
```

内置桥(全部零硬依赖,动态探测宿主对象):

- **webBridge**:localStorage、`window.open`、visibilitychange → `app:paused/resumed`。
- **telegramBridge**:直接读 `window.Telegram.WebApp`(不依赖 @twa-dev/sdk):
  `ready()+expand()` 初始化、`openLink`、`BackButton` → 发 `app:back`、
  `themeParams` 暴露为 `getTheme()`、viewportChanged → safe-area。
- **tauriBridge**:`storage()` 默认仍 localStorage(WebView 持久),提供
  `createTauriFileStorage()`(动态 `import('@tauri-apps/plugin-fs')`,壳模板装依赖)
  可选升级为文件存档;`openExternal` 走 shell 插件;窗口关闭 → `app:paused`。
- **capacitorBridge**:App 插件 pause/resume/backButton → 总线;
  safe-area 读 CSS env();Haptics 可用则接 vibrate。
- **weappBridge**:wx.onShow/onHide → 总线;storage() = adapters-weapp 的实现
  (由 adapters-weapp 注入注册,platform 不反向依赖它——通过
  `registerBridge(kind, factory)` 注册机制解耦)。

### 3.3 事件表扩展(declaration merging,与 environment/net 同机制)

```ts
'app:paused': {}          // 切后台/失焦/小程序 onHide
'app:resumed': {}
'app:back': {}            // Android 返回键 / TG BackButton(游戏决定关面板还是退出)
```

约定用法:audio 监听 `app:paused` 自动静音恢复(v1.1 在 audio 增加可选
`pauseOnHide: true` 配置,内部订阅这两个事件——仍只依赖 core)。

## 4. 新包 ②:@overworld-engine/adapters-weapp

微信环境适配器合集。依赖:仅 core。所有导出都是**注入物**,喂给既有 API:

```ts
createWeappStorage(): EnumerableStorage          // wx.getStorageSync/setStorageSync/
                                                 // removeStorageSync/getStorageInfoSync().keys
createWeappSocketTransport({ url, peerId? }): Transport   // wx.connectSocket 包装,
                                                 // 沿用 net 的 {from,data} 信封与重连语义
createWeappAudioBackend(): AudioBackend          // wx.createInnerAudioContext 映射(见 §5.1)
registerWeappBridge(): void                      // 向 platform 注册 weapp 桥
```

### 4.1 微信小游戏 3D(正式交付)

技术路径:R3F 的底层 `createRoot(canvas)`(**不依赖 react-dom**,官方自定义宿主 API)
+ 官方 weapp-adapter(window/document/XHR/RAF/触摸 polyfill,three.js 生态成熟路径)。
adapters-weapp 新增:

```ts
createWeappCanvasRoot(options?: {
  canvas?: WxCanvas            // 缺省 wx.createCanvas()
  dpr?: number                 // 缺省 wx.getSystemInfoSync().pixelRatio,clamp 2
}): { root: R3FRoot; canvas; size; dispose() }
// 内部:createRoot(canvas).configure({ gl 参数、size、events: 自定义触摸事件管理器 })
// + wx.onTouchStart/Move/End → R3F pointer 事件桥(射线拾取可用)

createWeappTouchJoystick(target: MovementInputRef, options?): { dispose() }
// 无 DOM 的摇杆:直接消费 wx 触摸事件,复用 input 包的纯函数摇杆数学
// (computeJoystickVector 等已导出),写入 Player 的 externalInput —— 键盘缺席时的移动方案
```

已知约束(写入 platforms.md 并在 API JSDoc 标注):
- drei `Text`(troika)不可用:标签用 scene 新增的 `SpriteLabel`(§5.3);
- `useGLTF` 走 adapter 的 XHR polyfill,模型必须放小游戏包内或 CDN 白名单域;
- weapp-adapter 版本在模板内锁定;基础库最低 2.19(WebGL1 兜底,three 自动降级)。

验证策略(**wx-shim harness**):在真实浏览器(playwright)注入模拟 `wx` 全局
(canvas/触摸/存储/socket 由真实浏览器能力支撑),完整跑 createWeappCanvasRoot +
SceneShell + Player 移动 + 任务链 E2E —— 适配层代码 100% 被真 WebGL 验证;
微信开发者工具真机预览作为最终人工确认(步骤文档化)。

单测策略:`vi.stubGlobal('wx', fakeWx)` 全量模拟,验证存储枚举、socket 信封
与重连、audio 后端行为对齐(与 HTMLAudio 路径同一组契约测试跑两遍)。


## 5. 既有包的增量(全部非破坏)

### 5.1 audio:后端注入
现状:内部直接 `new Audio(url)`。新增:

```ts
interface AudioBackend {
  create(url: string): AudioHandle   // AudioHandle:play/pause/volume/loop/
}                                    // onEnded/destroy 的最小结构接口
createAudioManager({ backend?: AudioBackend, pauseOnHide?: boolean, ... })
```

缺省 backend = HTMLAudio 实现(行为不变);weapp 传 `createWeappAudioBackend()`。
`pauseOnHide: true` 订阅 `app:paused/resumed`(默认 false,保持现状)。

### 5.2 input:触控默认值
`<VirtualJoystick>` 不改;游戏用 platform 的 `shouldShowTouchControls()` 决定挂载。
新增 `visible?: boolean` prop(纯便利,默认 true)。

### 5.3 scene:跨端标签 SpriteLabel(小幅新增)
新增 `<SpriteLabel text color? background? fontSize? />`:canvas 纹理 + three Sprite,
零 DOM/Worker 依赖,所有端可用。`BaseNPC`/`BaseBuilding` 新增
`labelMode?: 'troika' | 'sprite'`(默认 'troika' 保持现状;weapp 传 'sprite')。
交互提示气泡与任务徽章同机制切换。

### 5.4 net / core:零改动
画质推荐、存储、socket 的注入点在 v0.x 已备齐,这正是本设计能收敛到两个新包的原因。

## 6. 端模板(examples/)

四个模板,共同要求:只用公开 API、README(简体中文)含从零到真机/上架前的完整步骤、
壳 SDK 依赖只存在于模板内。

### 6.1 examples/telegram-mini-app
starter 玩法裁剪版 + `telegramBridge`:进入即 `ready/expand`、BackButton 关对话框
(订阅 `app:back`)、TG 主题色映射到 HUD、云存档演示用 `createRestStorage` 指向
游戏后端(文档说明 TG 的 CloudStorage 也可作为适配器接入,列为进阶)。
部署:复用 Ship Dock;README 给 BotFather 配置步骤。
**验收**:web E2E(playwright 注入 `window.Telegram.WebApp` mock)全绿。

### 6.2 examples/desktop-tauri(macOS + Windows)
Tauri 2 壳包 starter 构建:窗口配置(尺寸/标题/图标)、`createTauriFileStorage`
文件存档演示、Cmd+Q/关窗生命周期、自动更新留作文档。构建脚本:
`pnpm tauri:dev` / `pnpm tauri:build`(产 .app/.dmg 与 .msi/.exe)。
**验收**:macOS 本机 `tauri build` 出包 + 启动截图;Windows 出包步骤文档化
(本机无 Windows,CI 矩阵列为后续)。

### 6.3 examples/mobile-capacitor(iOS + Android)
Capacitor 7 壳:safe-area(viewport-fit=cover + env() padding 接 HUD)、
触控默认开摇杆、App 插件生命周期→总线、状态栏配置。
`pnpm cap:sync` 后 `npx cap open ios/android`。
**验收**:web 构建 + `cap sync` 双平台成功;真机步骤文档化(签名需开发者账号)。

### 6.4 examples/weapp-game(微信小游戏,完整 3D)
小游戏工程(game.js/game.json/project.config.json,esbuild 打包 npm 依赖):
weapp-adapter + `createWeappCanvasRoot` + SceneShell(sprite 标签)+ Player
(`createWeappTouchJoystick` → externalInput)+ 任务/对话/背包引擎 +
`createWeappStorage` 存档 + wx socket 联机 presence。玩法复用 starter 的水晶收集流程。
**验收**:wx-shim harness 下 playwright E2E(渲染 + 移动 + 任务链)全绿;
微信开发者工具预览步骤文档化(需登录,不进 CI)。
小程序(WXML 无头层)接法写入 platforms.md 指南(不单独立模板,能力由同一批适配器覆盖)。

## 8. 风险与对策

1. **微信小游戏 3D 兼容面**:R3F 依赖 scheduler 的 setTimeout 回退与 adapter polyfill,
   基础库变动有破坏风险 → 模板锁定 weapp-adapter 版本;wx-shim harness 进 CI 持续回归;
   drei 仅用已验证子集(GLTF/动画/数学组件),troika 文字明确禁用改用 SpriteLabel。
2. **WebView 性能(低端 Android)**:已有画质预设兜底;platform 的
  `recommendedQualityPreset()` 对 capacitor 默认压到 medium 以下。
3. **壳 SDK 版本漂移**(Tauri/Capacitor 大版本):模板锁定大版本并在 README 声明;
   框架包零依赖不受影响。
4. **审核合规**:TG/微信对外链、支付、热更新各有红线 → 集中写进 platforms.md,
   框架不内置任何越线能力。
