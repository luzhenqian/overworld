## v1.3 原生交互与出包

- **微信小游戏完整交互**:R3F 指针桥(`createWeappPointerBridge`,wx 触摸驱动的 raycast 拾取,点 NPC 网格→onClick)+ vendor `XMLHttpRequest`-over-`wx.request` polyfill 使 `useGLTF` 可加载真实 GLB;wx-shim harness 新增两条真断言
- **Telegram CloudStorage 云存档**:`createTelegramCloudStorage`(异步加载→同步 Map 镜像→异步写穿);**透明键编码**(`overworld:quest` ⇄ `overworld_003Aquest`)让冒号命名的框架 store 直接接入 Telegram 的 `[A-Za-z0-9_]` 键约束
- **CI 出包矩阵**:`ci.yml` 新增 examples 编译 job;`build-artifacts.yml` 产出无签名的 Tauri(macOS/Windows)+ Capacitor(Android debug / iOS 模拟器)构件;修复 `pnpm/action-setup` 与 `packageManager` 的冲突
- scene `SceneShell` 透传 `labelMode`;`createWeappCanvasRoot` 内置 `extend(THREE)`

## v1.2 确定性与联机基建

响应首个生产消费方(投资模拟游戏)的需求反馈:

- **确定性注入**:achievements/quest/saveSlots/presence 的 `clock`、toasts 的 `clock + scheduler`(默认不变;重放全等测试背书)
- **`@overworld-engine/relay`(新包)**:正式中继服务器(npx/编程 API/挂载既有 http server),与 net 客户端真 socket 互操作测试
- **线路协议规范**:信封/peerId/房间/`t` 多路复用/自建兼容中继检查单,2.0 前只增不改
- 权威多人接入指南(外部确定性内核作为共享 step)
- DOM 组件 `data-testid` + 官方测试指南(store 驱动断言);`configureEditorLabels` 文案覆写;持久化互操作指南;npm workspaces 兼容确认

## v1.1 多端支持

- **新包 `platform`**:平台检测/能力/桥(web/telegram/tauri/capacitor/weapp),`app:paused/resumed/back` 事件
- **新包 `adapters-weapp`**:wx 存储/socket/音频适配器、`createWeappCanvasRoot`(R3F 免 react-dom)、触摸摇杆
- **微信小游戏完整 3D**:examples/weapp-game(wx-shim 真浏览器 harness 21 断言,发布包体 1.01MB)
- 端模板:telegram-mini-app / desktop-tauri(macOS 实测出包)/ mobile-capacitor
- audio 后端注入 + `pauseOnHide`;scene `SpriteLabel` 跨端标签 + `labelMode`
- 指南:docs/guides/platforms.md(矩阵/接线/注入对照/上架红线)

## v0.9(破坏性变更收官)

- **P1** store 形态统一:dialogue/quest/audio 全部改为 `{ store, ...方法 }`(React 端 `useStore(engine.store, sel)`),全框架 8 个引擎形态一致
- **P2** `interact` → `entity:interact`(双发过渡,旧名 2.0 移除)
- **P3** audio persist 默认对齐约定(省略 = 关闭)
- **P11** `NPCWalker` 新增 `driven`/`tree`,行为树组合不再双步进
- **P12** BaseNPC/BaseBuilding 回退几何体与标签高度尊重 `scale`(默认值下视觉不变)
- **P13** 模型加载改为标准 Suspense + 错误边界;顺带修复 Player/Portal 无边界导致的潜在整树挂起
- **P14** `blockCell`/`createNavGridFromCells` 格子级建网格(tile 地图一等公民)

## v0.8

- **第二款示例游戏 `examples/dungeon`**:种子化程序地牢、行为树敌人(巡逻→追击→回岗)、钥匙宝箱任务链、HPA* 引路点、心心/击退/通关计时 —— 验证框架泛化性,并带回 6 个 1.0 前待修阻力点
- **1.0 API 冻结评审**(docs/api-review-1.0.md):就绪度 B+,15 项提案(store 形态统一、`interact` 加域名前缀等);低风险修复已落地(audio `events` 规范别名、persist 文档纠错、core README)
- **发布流水线**:changesets(fixed 版本组)+ CI/Release 工作流,18 包 `publish --dry-run` 全通过
- **性能基准**(benchmarks/):24 项基准 + 基线对比 + 8 个计数型回归守护;HPA* 访问格数少 17.9 倍,事件总线 ~250 万 emit/秒
